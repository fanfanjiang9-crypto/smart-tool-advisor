# 替代推荐引擎

## 推荐逻辑

推荐不是"找同类工具"，而是**基于使用者的场景缺口，找最优解**。

## 决策树

```
使用者指定了目标工具？
  │
  ├─ 有目标工具 → 评测结果是 Yes（综合 ≥ 3.5）？
  │                  ├─ 是 → 进阶推荐 + 互补推荐（各 1 个）
  │                  │        ├─ 进阶：同类但更强的工具（当你想更进一步时）
  │                  │        └─ 互补：弥补目标工具短板的工具
  │                  │
  │                  └─ 否 → 替代推荐（3 个）
  │                           ├─ 优先匹配使用者的核心痛点
  │                           ├─ 优先匹配使用者的预算
  │                           └─ 优先匹配使用者的技术水平
  │
  └─ 无目标工具 → 直接推荐（3-5 个工具组合）
                    ├─ 核心主力（1-2 个）：直接解决核心痛点，优先免费
                    ├─ 辅助配套（1-2 个）：弥补主力短板，补齐完整工作流
                    └─ 进阶升级（1 个）：当前够用但未来值得关注
```

## 推荐优先级规则

1. **痛点匹配优先**：先看哪个工具最能解决使用者的核心痛点，再看价格、上手难度
2. **预算硬约束**：不推荐超出使用者预算的工具，除非标注"超预算但值得考虑"
3. **门槛匹配**：不推荐使用者技术水平无法驾驭的工具，除非标注"需要学习但回报高"
4. **避免推荐全家桶**：使用者说缺一个螺丝刀，不要推荐一整套工具箱
5. **开源偏好匹配**：如果使用者明确要求开源，优先推荐开源工具；闭源工具只有在明显更优时才推荐，且必须标注"闭源替代方案"

## 推荐信息标准

每个推荐工具必须包含：

| 字段 | 说明 |
|------|------|
| 工具名 | 全名 + 一句话定位 |
| 开源/闭源 | 🟢 开源 / 🟡 半开源 / 🔴 闭源 / ⚪ 免费但闭源 + GitHub 地址（如开源） |
| 为什么更适合 | 直接关联使用者画像中的痛点 |
| 和原工具的核心差异 | 哪里比原工具更好/哪里不如 |
| 价格 | 免费额度 + 付费版价格 |
| 从哪里开始 | 推荐一个具体的入门资源 |
| 一句话警告 | 这个工具最大的坑是什么 |

## 常见场景的推荐方向

> 以下不是硬规则，是基于场景的启发式推荐方向。实际推荐必须结合具体使用者画像。

### 内容创作者 → 优先推荐
- 选题调研：Perplexity / Deep Research / Tavily
- 内容撰写：Claude / ChatGPT / 通义千问
- 配图生成：Midjourney / DALL-E / 可画 AI / 即梦
- 多平台分发：各平台后台 + 定时发布工具
- 数据复盘：各平台分析后台 + 新榜 / 蝉妈妈

### 技术开发者 → 优先推荐
- 编码助手：Claude Code / Cursor / GitHub Copilot
- 项目管理：Linear / Notion / Plane
- 文档协作：Notion AI / Mintlify
- 自动化：n8n / Make / Zapier

### 学习研究者 → 优先推荐
- 信息筛选：Perplexity / Glarity / 简悦
- 深度阅读：Readwise / Omnivore / Cubox
- 笔记管理：Obsidian / Notion / Logseq
- 输出辅助：Claude（写摘要、做思维导图）

### 商务运营者 → 优先推荐
- 文档处理：ChatGPT / Claude + Office 插件
- 数据分析：ChatGPT Advanced Data Analysis / Julius AI
- 流程自动化：飞书多维表格 + AI / 钉钉 AI
- CRM：HubSpot AI / Salesforce Einstein
